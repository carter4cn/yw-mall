## 今天的核心修复

### `mall-common/configcenter/loader.go`：yaml.Unmarshal → conf.LoadFromYamlBytes

**症状**：所有 RPC 启动后 `Etcd.Hosts` 为空，导致服务**不向 etcd 注册自己**，
mall-api 通过 etcd discovery 找不到任何 RPC，调用全挂。

**根因**：`yaml.v3` 在多层匿名嵌入结构上**静默丢字段**：

```
Config (用户)
  └── service.ServiceConf  (anonymous embed, go-zero)
        └── discov.EtcdConf
              └── Hosts []string  ← 丢
```

`yaml.Unmarshal` 解析进 `dest any` 时只填了顶层字段；嵌套层的 `Hosts/Key`
等字段没传下去。这同样影响 `RestConf.Host`、`RestConf.MinIO.Endpoint` 等
go-zero 内部 embed 的字段。

**修法**：换成 go-zero 自家的 `conf.LoadFromYamlBytes`，它认识嵌入结构和
`json:",default=..."` 标签，按 `mapping → reflect.Set` 路径正确赋值。

附带 `go mod tidy`：
- `mall-common/go.mod` 移除 `gopkg.in/yaml.v3` 直接依赖
- `mall-review-rpc/go.mod`、`mall-activity-async-worker/go.{mod,sum}` 同步
- `mall-activity-async-worker/go.mod` 新增 indirect: `pelletier/go-toml/v2`、`titanous/json5`、`yaml.v2`（go-zero conf 间接依赖）
