## 踩到的坑

### 1. `review_admin.sql` 用了 MariaDB-only 语法

```sql
ALTER TABLE `review` ADD COLUMN IF NOT EXISTS `shop_id` bigint NOT NULL DEFAULT 0;
CREATE INDEX IF NOT EXISTS `idx_shop_id` ON `review`(`shop_id`, `status`, `id`);
```

MySQL 8 不支持 ALTER 和 CREATE INDEX 上的 `IF NOT EXISTS`，导致 P1 上线时 `review_delete_request` 表静默没建出来。**修法**：拆成普通 `ALTER TABLE ... ADD COLUMN` + `ADD INDEX`；今后所有 DDL 走 plain 语法，幂等性靠 db-init 流程外层保证。

### 2. F-5 测试时读不到刚写的 credit_score

```go
// 旧版
UPDATE shop SET credit_score = GREATEST(0, LEAST(200, credit_score+?)) WHERE id=?
SELECT credit_score FROM shop WHERE id = ?
```

UPDATE 走 ProxySQL 主，SELECT 走从库，复制还没追上 → 读到旧值，阈值 switch 永远命中 default 分支，看起来"没生效"。

**修法**：read-then-write（一次 SELECT + 进程内计算 next + 一次 UPDATE），把 next 直接传给 `applyThresholdRules`，**永远不再回库读最新值**。

### 3. proto 生成 + 客户端封装的人工同步

go-zero goctl 不自动给 client 写新方法。每次改完 `mall-common/proto/<svc>/<svc>.proto`：

```bash
export PATH=/home/carter/workspace/go/bin:$PATH
cd mall-<name>-rpc
protoc --go_out=. --go-grpc_out=. --proto_path=. --proto_path=../mall-common/proto \
       ../mall-common/proto/<name>/<name>.proto
# 然后手动改 <name>service/<name>service.go 接口 + 实现，新方法逐个加
```

漏改 client 就会出现 "all build OK 但 admin-api 调不到新 RPC"。值得在 README 里写死流程（已加到 yw-mall-admin/README.md）。
